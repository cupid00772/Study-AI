# 台灣聯合大學系統 105學年度學士班轉學生考試試題 詳解
**科目：計算機概論 (A4 804)**

## 單選題 (1~20)

**1. (B)**
- **解析：** 將十進位 $19.375$ 轉為二進位。
  整數部分：$19 = 16 + 2 + 1 = 10011_2$
  小數部分：$0.375 = 0.25 + 0.125 = 2^{-2} + 2^{-3} = 0.011_2$
  合併為 $10011.011_2$。

**2. (C)**
- **解析：** 8-bit 有號整數相加。
  $X = 01100001_2 = 97$
  $Y = 00001111_2 = 15$
  $X + Y = 112 = 01110000_2$。
  或者直接進行二進位加法：`01100001 + 00001111 = 01110000`。

**3. (C)**
- **解析：** 判斷哪一個電路「不是」2-input XNOR gate。
  (A) XOR gate 接 NOT gate，即為 XNOR。
  (B) 直接是 XNOR 的符號。
  (C) 這是 XOR 的符號，所以不等於 XNOR。
  (D) $(A \cdot B) + (A' \cdot B')$，這是 XNOR 的布林代數式，等於 XNOR。

**4. (E) 或 (A)**
- **解析：** IEEE 754 單精度浮點數加法。
  $X$: Sign=0, Exp=129 ($2^2$), Mantissa=$1.0111_2$
  $Y$: Sign=0, Exp=134 ($2^7$), Mantissa=$1.0100001111_2$
  為了相加，將小階數對齊大階數 ($X$ 右移 5 位)：
  $X$ 的 Mantissa 變為 $0.000010111_2$ (此時 Exp 變為 134)
  相加：$1.0100001111_2 + 0.000010111_2 = 1.0100111101_2$
  結果應為：Sign=0, Exp=10000110, Fraction=010011110100...
  對照選項，(A) 為 `0 10000110 01001110100...`，與正確計算結果有些微出入（可能是題目出錯）。嚴格來說答案是 (E) None of the above，但若考慮出題瑕疵，可能會給 (A)。

**5. (D)**
- **解析：** 作業系統透過對行程的資源限制 (resource restrictions) ，例如打破 Circular Wait 等條件，可以防止「死結 (Deadlock)」的發生。

**6. (E)**
- **解析：** 關於抽象資料型別 (ADT)。
  (A) 錯誤，許多語言 (如 C++, Java) 有提供 ADT 套件 (如 STL, Collections)。
  (B) 錯誤，由一端加入、另一端取出的線性結構稱為 Queue，非 Stack。
  (C) 錯誤，排隊等公車是 FIFO，屬於 Queue，非 Stack。
  (D) 錯誤，Queue 是 FIFO (First-in, First-out) 結構，不是 LIFO。
  因此皆不正確，選 (E)。

**7. (D)**
- **解析：** 關於檔案結構。
  (A) 錯誤，indexed 和 hashed 都可以 randomly access。
  (B) 錯誤，從頭到尾讀取應該使用 sequential access。
  (C) 錯誤，使用數學函數將 key 映射到位址的是 hashed file，而非 indexed file。
  因此選 (D)。

**8. (B)**
- **解析：** TCP/IP 協定模型中，Transport layer 是 Layer 4，Network layer 是 Layer 3。選項 (B) 寫反了，所以不正確。

**9. (B)**
- **解析：** 負責在實體媒介上傳輸位元串流 (bit stream) 的是 Physical layer (實體層)。

**10. (B)**
- **解析：** 簡化布林代數：
  $(P+Q'+R') \cdot (P+Q'+R) \cdot (P+Q+R')$
  前兩項合併：$(P+Q') + R' \cdot R = P+Q'$
  再與第三項相乘：$(P+Q') \cdot (P+Q+R') = P + Q'(Q+R') = P + Q'R'$
  對應選項 (B)。

**11. (D)**
- **解析：** Class C 網路的預設子網路遮罩是 `255.255.255.0`。若要切割 (separate/subnet)，必須向 Host ID 借位，遮罩最後一個 byte 必須為連續的 1，如 128, 192, 224 等。選項中只有 `255.255.255.192` (借 2 bits) 是合法的切割。

**12. (A)**
- **解析：**
  `int a=12; int *c = &a;` 此時 c 指向 a，*c 為 12。
  `*c = *c + 3;` 相當於 `a = a + 3;`，所以 a 變成 15。
  印出的順序為：先印出原本的 c=12 a=12，計算後印出 c=15。
  所以結果是 12 12 15。

**13. (E)**
- **解析：** 
  `int a=17; int *c=&a;`
  `*c = *c + 3;` -> a 變為 20
  `a = a + 5;` -> a 變為 25
  最後印出 a 的值，為 25。

**14. (C)**
- **解析：** 
  `a = 10, b = 4.2` (float)
  `(int)(a/b)`：10 / 4.2 = 2.38095...，轉為 int 取整數為 2。
  `printf("%f", a/b)`：浮點數預設印出小數點後六位，經過浮點運算精確度影響，結果印出 `2.380953`。
  故選 (C)。

**15. (B)**
- **解析：** `aPtr` 是一個指標指向 `a` 的位址。因此 `*aPtr` 的值等於 `a` 的值，也就是 21。`(B) a=21 and *aPtr=21` 是完全正確的描述。

**16. (D)**
- **解析：** 巢狀迴圈：
  i=0 時，j=0,1,2，印出 0*0, 0*1, 0*2 -> `0 0 0`
  i=1 時，j=0,1,2，印出 1*0, 1*1, 1*2 -> `0 1 2`
  i=2 時，j=0,1,2，印出 2*0, 2*1, 2*2 -> `0 2 4`
  符合選項 (D)。

**17. (B)**
- **解析：** 程式碼內迴圈 `for (j = n; j > i; j--)` 會將較小的值如同泡泡般往前交換 (`data[j-1] > data[j]` 則 swap)，這是標準的氣泡排序 (Bubble Sort) 演算法。

**18. (C) 或 (E)**
- **解析：** `f1(n)` 是遞迴計算從 1 加到 n 的總和。
  `f1(6) = 1+2+3+4+5+6 = 21`，所以 (C) 敘述正確。
  `f1(8) = 1+...8 = 36`，所以 (E) 敘述也正確。
  本題可能有多選含意，或題目給分標準較寬，兩者皆為正確結果。

**19. (A)**
- **解析：** 
  `f2` 使用輾轉相除法求最大公因數 (GCD)。
  `f1` 傳回 `(x * y) / GCD(x, y)`，由數學關係可知，這是在求兩數的最小公倍數 (LCM)。

**20. (A)**
- **解析：** 
  這是一個遞迴版本的輾轉相除法，用於求最大公因數 (GCD)。當餘數 `n == 0` 時，傳回目前的被除數 `m`。

  (E) 對稱式加密實務上常透過 KDC (Key Distribution Center) 來分發與管理金鑰。正確。

**22. (B), (C), (E)**
- **解析：** 題目要求選「不正確 (not correct)」的。
  (A) 正確敘述。
  (B) 讀寫頭可以向左也可以向右移動。故不正確。
  (C) 圖靈機是理論模型，非 IBM 擁有的超級電腦。故不正確。
  (D) 正確敘述。
  (E) 停機問題 (Halting Problem) 已被證明是圖靈機無法解決 (Undecidable) 的問題。故不正確。

**23. (A), (B), (C), (D)**
- **解析：** 根據樹狀圖，該運算式樹的根為 `+`，左子樹為 `a * (b+c)`，右節點為 `d`。
  (A) Infix: `a * (b+c) + d` (正確)。
  (B) Prefix: `+ * a + b c d` (正確)。
  (C) Postfix: `a b c + * d +` (正確)。
  (D) 該樹中每個節點最多兩個子節點，屬於二元樹 (Binary Tree) (正確)。
  故 ABCD 皆為正確敘述。

**24. (A), (E)**
- **解析：**

  (E) Prime area (主要區) 是雜湊表中一開始存放資料的區塊，發生碰撞時才可能移至溢出區 (Overflow area)。正確。
